# はじめに

## 本書の目的

本書は、GoF（Gang of Four）が提唱した23のデザインパターンを、Go言語（Golang）のサンプルコードを通じて学ぶことを目的としています。

デザインパターンは1994年に出版された「Design Patterns: Elements of Reusable Object-Oriented Software」で紹介されて以来、ソフトウェア設計における重要な知識体系となっています。

## デザインパターンとは

デザインパターンとは、ソフトウェア設計において繰り返し現れる問題に対する、再利用可能な解決策のことです。パターンは以下の要素で構成されます：

- **パターン名**: 問題と解決策を簡潔に表す名前
- **問題**: どのような状況でパターンを適用するか
- **解決策**: 設計の構成要素と関係性
- **結果**: パターン適用のトレードオフ

## GoFパターンの分類

GoFの23パターンは、目的によって3つのカテゴリに分類されます：

### 生成に関するパターン（Creational Patterns）

オブジェクトの生成に関するパターンです。

| パターン名 | 概要 |
|-----------|------|
| Singleton | インスタンスが1つだけ存在することを保証 |
| Factory Method | オブジェクト生成をサブクラスに委譲 |
| Abstract Factory | 関連するオブジェクト群を生成 |
| Builder | 複雑なオブジェクトを段階的に構築 |
| Prototype | 既存オブジェクトをコピーして新規作成 |

### 構造に関するパターン（Structural Patterns）

クラスやオブジェクトの構成に関するパターンです。

| パターン名 | 概要 |
|-----------|------|
| Adapter | インターフェースを変換して互換性を確保 |
| Bridge | 抽象と実装を分離 |
| Composite | 部分-全体の階層構造を表現 |
| Decorator | 動的に機能を追加 |
| Facade | サブシステムへの統一インターフェース |
| Flyweight | 多数のオブジェクトを効率的に共有 |
| Proxy | 別オブジェクトへのアクセスを制御 |

### 振る舞いに関するパターン（Behavioral Patterns）

オブジェクト間の責務分担と通信に関するパターンです。

| パターン名 | 概要 |
|-----------|------|
| Chain of Responsibility | リクエストを処理者の連鎖に渡す |
| Command | リクエストをオブジェクトとしてカプセル化 |
| Interpreter | 言語の文法を解釈・実行 |
| Iterator | 要素への順次アクセスを提供 |
| Mediator | オブジェクト間の通信を仲介 |
| Memento | オブジェクトの状態を保存・復元 |
| Observer | 状態変化を依存オブジェクトに通知 |
| State | 状態に応じて振る舞いを変更 |
| Strategy | アルゴリズムをカプセル化して交換可能に |
| Template Method | アルゴリズムの骨格を定義 |
| Visitor | 構造を変えずに新しい操作を追加 |

## なぜGoでデザインパターンを学ぶのか

Go言語は従来のクラスベースのオブジェクト指向言語（Java、C++など）とは異なるアプローチを取ります：

- **クラスがない**: 構造体（struct）とメソッドで表現
- **継承がない**: 埋め込み（embedding）とインターフェースで実現
- **暗黙的インターフェース**: インターフェースの実装を明示的に宣言しない

このような特性を持つGoでデザインパターンを学ぶことで：

1. **パターンの本質を理解できる**: クラスベースの実装に縛られない、パターンの核心を理解できます
2. **Goの特性を活かせる**: Goらしいイディオマティックな実装方法を学べます
3. **実践的なスキルが身につく**: 実務で使える設計力とGo力が同時に向上します

## 本書の構成

各パターンの章は以下の構成で説明します：

1. **概要**: パターンの目的と適用場面
2. **構造**: パターンの構成要素（UMLまたは図解）
3. **実装**: Goによるサンプルコード
4. **使用例**: 実際のユースケース
5. **ポイント**: Goで実装する際の注意点

## 対象読者

- Go言語の基本文法を理解している方
- デザインパターンを学びたい方
- オブジェクト指向設計の考え方を身につけたい方
- より良いGoコードを書きたい方

次章では、本書を読み進めるために必要なGoの基礎知識を確認します。
